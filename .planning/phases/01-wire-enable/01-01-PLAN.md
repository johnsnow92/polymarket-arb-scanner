```python
# Attach fee path hints for cross-all opportunities
for opp in opportunities:
    pa = opp.get("_platform_a", "")
    pb = opp.get("_platform_b", "")
    if not pa or not pb:
        continue
    prices_str = opp.get("prices", "")
    a_yes = a_no = b_yes = b_no = 0.0
    for part in prices_str.split():
        if "=" not in part:
            continue
        key, val = part.split("=", 1)
        v = float(val)
        prefix = key.split("_")[0].lower()
        # Map prefix back to platform (e.g., "PO" -> polymarket, "KA" -> kalshi)
        # The type string format is Cross(PA-PB) where PA/PB are 2-char uppercase
        if key.endswith("_Y"):
            if key.startswith(pa[:2].upper()):
                a_yes = v
            else:
                b_yes = v
        elif key.endswith("_N"):
            if key.startswith(pa[:2].upper()):
                a_no = v
            else:
                b_no = v
    platforms = [pa, pb]
    yes_p = {pa: a_yes, pb: b_yes} if a_yes and b_yes else {}
    no_p = {pa: a_no, pb: b_no} if a_no and b_no else {}
    if yes_p and no_p:
        fee_path = find_lowest_fee_path(platforms, yes_p, no_p)
        if fee_path:
            opp["_fee_path"] = fee_path
```

IMPORTANT: The prices_str format from scan_cross_all is `"{pa}_Y={a_yes:.3f} {pb}_N={b_no:.3f}"` (line 437) or `"{pa}_N={a_no:.3f} {pb}_Y={b_yes:.3f}"` (line 440). The prefix uses the first 2 chars of the platform name uppercased. So for polymarket vs kalshi: `"PO_Y=0.400 KA_N=0.550"`. Parse accordingly — split on `_Y=` and `_N=` to extract the platform prefix and price. We need all 4 prices (both YES and NO for both platforms) but the opp only stores the 2 prices of the winning strategy. For the `_fee_path` hint, use the stored prices as-is for the platforms available; for the missing prices, we can still call `find_lowest_fee_path` with just the 2 available prices by setting the "wrong side" prices to 0 (they'll be filtered out by the `<= 0` check in `find_lowest_fee_path`). This means the fee path will only evaluate the detected strategy's routing, not discover an alternative strategy — which is correct behavior for a hint.
