Create `scans/correlated.py` with two-stage detection for correlated market divergences.

**Stage 1 Function: `scan_correlated()`**
- Input: `markets_by_key: dict, correlated_pairs: list[tuple], min_spread: float = 0.10, price_cache: dict | None = None`
- Receives config: `correlated_pairs = [("Bitcoin $100k", "Bitcoin $90k"), ("Ethereum $5k", "Ethereum $4k"), ...]` — market titles or IDs
- For each pair in correlated_pairs:
  - Find both markets in markets_by_key (via fuzzy match on title if needed, or direct ID lookup)
  - Get mid prices for both markets via price_cache or API call
  - Calculate spread: `spread = abs(price_a - price_b) / max(price_a, price_b)`
  - If spread >= min_spread threshold (default 0.10 = 10%):
    - Determine which is underpriced vs overpriced
    - Return opportunity with: `type: "Correlated"`, `market_a`, `market_b`, `price_a`, `price_b`, `spread`, `_long_leg` (underpriced), `_short_leg` (overpriced), `_token_ids_a`, `_token_ids_b`
- Log: `"Correlated pair: %s vs %s, spread=%.1f%%", market_a, market_b, spread*100`

**Helper: `_calculate_spread(price_a: float, price_b: float) -> float`**
- Formula: `abs(price_a - price_b) / max(price_a, price_b)`
- Handle division by zero (return 0.0)
- Returns spread as decimal (0.10 = 10%)

**Stage 2 Function: `_refine_correlated_with_depth()`**
- Input: `opportunities: list[dict]`
- Re-validates both legs still exist and have sufficient liquidity
- Fetches current CLOB depths for both legs
- Drops if either leg no longer has depth >= MIN_LIQUIDITY (from config)
- Drops if spread has collapsed by >20% (arbitrage opportunity closed)
- Returns refined list
- Log: `"Correlated refined: %d/%d pairs have sufficient liquidity", len(refined), len(opps)`

**Config Helper: `_load_correlated_pairs(config_json: str) -> list[tuple]`**
- Input: JSON string like `'[["Bitcoin $100k", "Bitcoin $90k"], ["Eth $5k", "Eth $4k"]]'`
- Parse JSON and validate format (list of 2-tuples)
- Return list[tuple[str, str]]
- Raise ValueError if JSON malformed or pairs not 2-tuples

**Integration note:** Will be called from executor.py _build_legs() dispatcher. Do NOT add executor integration yet.

```python
def net_profit_correlated(
    long_entry_price: float,
    long_exit_price: float,
    short_entry_price: float,
    short_exit_price: float,
    size: float,
    platform_long: str = "polymarket",
    platform_short: str = "polymarket",
) -> float:
```

**Logic:**
- Correlated trades are Layer 4 (informed), typically both legs on Polymarket or Kalshi
- Both legs incur taker fees (time-sensitive execution)
- Long leg: buy at long_entry_price, sell at long_exit_price
  - Profit = size * (long_exit_price - long_entry_price) - fees
- Short leg: sell at short_entry_price, buy at short_exit_price (reversed for NO leg)
  - Profit = size * (short_entry_price - short_exit_price) - fees
- For Polymarket: `fee_rate = POLYMARKET_TAKER_FEE_RATE` for both legs
- For Kalshi: `fee_rate = 0.02 * KALSHI_MAKER_MULTIPLIER` for both legs
- Total net = long_profit + short_profit

**Docstring:**
- Explain: "Correlated pair convergence trades match-size both legs (long underpriced, short overpriced). Both legs pay taker fees."
- Parameters: long_entry/exit (YES leg prices), short_entry/exit (NO leg prices), size (same for both), platforms (can differ)
- Return: Net profit USD (may be negative)

Match existing fee function style (minimal validation, assume callers validate inputs).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config JSON → correlated pair list | Untrusted config from env var; must validate JSON format and tuple structure |
| Market data → spread calculation | Price data from API; must validate prices are in [0, 1] range |
| Spread signal → trade execution | Spread divergence does not guarantee convergence; Stage 2 validation confirms both legs available |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Tampering | Correlated pair config injection | mitigate | Validate JSON structure and tuple format in _load_correlated_pairs(); reject malformed config |
| T-08-07 | Spoofing | One leg unavailable at execution | mitigate | Stage 2 refinement checks both legs have depth >= MIN_LIQUIDITY; drops if either leg missing |
| T-08-08 | Information Disclosure | Correlation config logged | mitigate | Log only market IDs/titles, not full config; never log price data |
| T-08-09 | Denial of Service | API calls for all pairs on every scan | mitigate | Cache prices in memory with 60s TTL; avoid redundant API calls per pair |
| T-08-10 | Elevation of Privilege | Unvalidated config enables trades on wrong markets | mitigate | CORRELATION_DIVERGENCE_THRESHOLD validated in config.py; default 0.10 is conservative; CLI accepts --mode correlated only if enabled |

</threat_model>
