Create `scans/time_decay.py` with two-stage detection for near-expiry high-consensus opportunities.

**Stage 1 Function: `scan_time_decay()`**
- Input: `markets_by_key: dict, signal_aggregator, min_hours_to_expiry: int = 48, min_consensus: float = 0.90, buy_below_price: float = 0.95, price_cache: dict | None = None`
- For each market:
  - Check time to resolution: `hours_left = (resolution_timestamp - now) / 3600`
  - Skip if hours_left > min_hours_to_expiry OR hours_left < 1 (not in sweet spot)
  - Get consensus probability via `signal_aggregator.get_consensus(market_key)` (returns float 0-1, or None)
  - Skip if consensus is None or < min_consensus
  - Determine consensus side: YES if consensus > 0.50, else NO
  - Calculate target price: consensus if YES side, (1 - consensus) if NO side
  - If target_price < buy_below_price:
    - Return opportunity with: `type: "TimeDecay"`, `market`, `_hours_to_expiry`, `_consensus_side` (YES/NO), `_consensus_prob`, `_target_price`, `_guaranteed_gain` (buy_below_price - target_price)
- Log: `"TimeDecay: %s, expiry=%.1f hours, consensus=%.1f%%, gain=%.1f%%", market, hours_left, consensus*100, guaranteed_gain*100`

**Helper: `_check_time_to_expiry(resolution_timestamp: int, min_hours: int = 48) -> float | None`**
- Input: resolution_timestamp (Unix seconds), min_hours threshold
- Returns: hours left as float, or None if outside sweet spot (too early or too late)
- Threshold: 1 < hours_left <= min_hours
- Log warnings if timestamp is in past or too far in future

**Helper: `_validate_consensus(consensus: float, min_threshold: float = 0.90) -> bool`**
- Input: consensus probability (0-1), min_threshold
- Returns: True if consensus >= min_threshold, False otherwise
- Also check sample size (optional): if signal_aggregator provides sample_count, require >30 forecasters (mitigates small-sample overconfidence)

**Stage 2 Function: `_refine_time_decay_with_prices()`**
- Input: `opportunities: list[dict]`
- Re-validates current market prices (may have moved since Stage 1)
- Drops if current price has risen significantly above buy_below_price (opportunity closed)
- Drops if hours_to_expiry has dropped below 1 hour (too late to enter)
- Returns refined list
- Log: `"TimeDecay refined: %d/%d still profitable at current price", len(refined), len(opps)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Market resolution timestamp → expiry calculation | Untrusted timestamp from API; must validate not in past or far future |
| Consensus probability → buy decision | Consensus not infallible; sudden events shift probability; Stage 2 validates before execution |
| Market price at resolution → position P&L | Resolution may surprise (event changes); position has no exit option (hold-to-resolution design) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-11 | Tampering | Resolution timestamp manipulation | mitigate | Validate timestamp is within [now, now + 365 days]; reject outliers; log suspicious timestamps |
| T-08-12 | Repudiation | Consensus shift before resolution | accept | Consensus (Metaculus + Manifold) is best-effort forecast; sudden events can invalidate; MAX_TIME_DECAY_POSITION = $50 limits loss per trade |
| T-08-13 | Information Disclosure | Market resolution leaks | accept | Resolution timestamps are public market data; no sensitive information exposed |
| T-08-14 | Denial of Service | Signal aggregator API call per market | mitigate | Cache consensus with 60s TTL; avoid redundant calls in same scan; use lazy evaluation |
| T-08-15 | Elevation of Privilege | Unvalidated feature flag enables position without exit | mitigate | TIME_DECAY_ENABLED defaults false; validate_config() checks thresholds in range; executor requires risk_manager approval |

</threat_model>
