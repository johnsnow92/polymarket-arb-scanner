**Stage 1 Function: `scan_news_snipe()`**
- Input: `markets_by_key: dict, finnhub_client: FinnhubNewsClient, cooldown_cache: dict`
- Fetches recent company news via `finnhub_client.fetch_company_news(symbol, from_date, to_date)`
- For each headline, calls `extract_news_signals(headlines, markets_by_key)` (see below)
- Filters by cooldown: skip market if cooldown_cache[market_key] > time.time()
- Returns opportunities with: `type: "NewsSnipe"`, `market`, `_headline`, `_sentiment` (YES/NO), `_confidence`, `_market_key`
- Log: `"NewsSnipe: matched %d headlines to %d markets", len(headlines), len(opps)`

**Helper: `extract_news_signals(headlines: list[dict], markets_by_key: dict) -> list[dict]`**
- Takes list of headlines from Finnhub and market dict
- For each headline, tries fuzzy matching headline text to market questions
- Uses `fuzz.token_set_ratio(headline_text, market_question) > FUZZY_MATCH_THRESHOLD` (default 70)
- Calls `_score_sentiment(headline_text)` to detect keywords and assign sentiment
- Returns signals with `_sentiment`, `_confidence` (0.0-1.0 score)

**Helper: `_score_sentiment(headline_text: str) -> dict`**
- Input: full headline string
- Define sentiment keywords:
  - YES keywords: ["approved", "confirmed", "passed", "granted", "successful", "adopted", "launched", "completed"]
  - NO keywords: ["rejected", "failed", "denied", "blocked", "withdrawn", "cancelled", "delayed"]
- Search headline for keywords; return first match
- Confidence scoring: keyword in headline text (not just headline) = 0.8, elsewhere in article = 0.6
- Return dict: `{"sentiment": "YES"|"NO"|None, "confidence": 0.0-1.0}`
- Log keyword matches: `"Sentiment matched: %s in '%s'", keyword, headline_text[:100]`

**Stage 2 Function: `_refine_news_with_confidence()`**
- Input: `opportunities: list[dict]`
- Filters opportunities where confidence < 0.5 (low confidence is rejected)
- Checks cooldown_cache[market_key] to prevent duplicate execution within 30s
- Returns refined list
- Log: `"News refined: %d/%d passed confidence threshold", len(refined), len(opps)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Finnhub API response → headline parsing | External API provides untrusted headlines; must validate structure and sanitize before fuzzy matching |
| News headline → market matching | Fuzzy matching may produce false positives; low-confidence matches execute unnecessary trades |
| Sentiment keywords → trading signal | Keyword presence in headline does not guarantee event resolution; confidence threshold mitigates |
| Cooldown cache → duplicate prevention | In-memory cache may be lost on crash; 30s cooldown provides short-term protection but multi-process systems need persistent cooldown (Redis) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Disclosure | Finnhub API key in logs | mitigate | Never log raw API responses, auth headers, or API key; use env var (FINNHUB_API_KEY), not hardcoded |
| T-08-02 | Tampering | Malicious headline from API | mitigate | Fuzzy match threshold 70 reduces false positives; confidence scoring requires keyword presence; tokens_set_ratio only matches semantically similar text |
| T-08-03 | Spoofing | Fake news trigger duplicate orders | mitigate | 30s cooldown per market prevents immediate re-execution; same headline within 30s is rate-limited |
| T-08-04 | Repudiation | News source authenticity | accept | Finnhub is trusted provider; we don't verify news authenticity independently (acceptable for trading bot) |
| T-08-05 | Denial of Service | Finnhub API rate limit exhaustion | mitigate | Use tenacity with exponential backoff (max 3 retries); free tier 60/min respected; WebSocket unlimited |
| T-08-06 | Elevation of Privilege | Unvalidated config enables trades outside policy | mitigate | Feature flag NEWS_SNIPE_ENABLED defaults to false; validate_config() (Phase 5) checks FINNHUB_API_KEY exists if enabled |

</threat_model>
