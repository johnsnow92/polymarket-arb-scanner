**Stage 1 Function: `scan_imbalance()`**
- Input: `markets_by_key: dict, min_ratio: float = 3.0, price_cache: dict | None = None`
- Iterates over Polymarket markets with `clobTokenIds`
- Calls `_calculate_imbalance_ratio(order_book, top_levels=5)` on YES token order book
- Returns opportunities with keys: `type: "Imbalance"`, `market`, `_imbalance_ratio`, `_direction` (YES/NO), `_token_ids`
- Log: `"Imbalance found: %s, ratio=%.2f, direction=%s", market, ratio, direction`

**Helper: `_calculate_imbalance_ratio(order_book, top_levels=5)`**
- Takes CLOB order book dict with `bids` and `asks` arrays
- Sums top N levels: `bid_vol = sum(bid["size"] for bid in bids[:top_levels])`
- Formula: `(bid_vol - ask_vol) / (bid_vol + ask_vol)` range [-1, 1]
- Returns 0.0 if no bids/asks
- Positive ratio = bid dominance (predict YES), negative = ask dominance (predict NO)

**Stage 2 Function: `_refine_imbalance_with_clob()`**
- Input: `opportunities: list[dict]`
- Re-fetches order book for each opportunity token ID
- Re-calculates imbalance ratio
- Drops if ratio collapsed by >30% (current ratio < 0.7 * original ratio magnitude)
- Returns refined list
- Log: `"Imbalance refined: %d/%d passed", len(refined), len(opportunities)`

**Integration note:** This module will be called from executor.py _build_legs() dispatcher (wired in Plan 5). Do NOT add executor integration yet — this task is scan-only.

```python
def net_profit_imbalance(
    entry_price: float,
    exit_price: float,
    size: float,
    platform: str = "polymarket",

```

**Logic:**
- Imbalance trades are Layer 4 (informed trading), typically on Polymarket or Kalshi
- Use taker fees (not maker) because signal-based execution is time-sensitive
- For Polymarket: `net = size * (exit_price - entry_price) - size * entry_price * POLYMARKET_TAKER_FEE_RATE - size * exit_price * POLYMARKET_TAKER_FEE_RATE`
- For Kalshi: Use KALSHI_MAKER_MULTIPLIER since Kalshi doesn't have taker/maker distinction; fee = size * entry_price * 0.02 * KALSHI_MAKER_MULTIPLIER
- Return max(net, 0) if you want to cap at zero; or return raw net profit

**Docstring:**
- Explain: "Imbalance signal execution uses taker fees due to time-sensitive nature. Entry and exit both incur fees."
- Parameter constraints: `0 < entry_price < 1.0`, `0 < exit_price < 1.0`, `size > 0`, `platform in ["polymarket", "kalshi", "gemini", "betfair", ...]`
- Return: Net profit in USD (may be negative)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
