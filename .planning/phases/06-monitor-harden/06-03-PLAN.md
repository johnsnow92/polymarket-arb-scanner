1. In `alerting.py`, in the `AlertManager.__init__()` method:
   - Add field: `self._strategy_losses: dict[str, deque] = {}` (import deque from collections at top if not already)
   - Add field: `self._strategy_opp_count: dict[str, int] = {}` (for zero-opp tracking)
   - Add field: `self._strategy_last_opp_time: dict[str, float] = {}` (for zero-opp window calculation)
   - Docstring update: "Loss streak tracking initialized per strategy"

2. Add method `check_strategy_loss_streak(self, strategy_type: str, trade_won: bool) -> bool`:
   - Input: strategy name (e.g. "binary", "cross", "kalshi") and whether trade was profitable (trade_won=True/False)
   - If strategy_type not in self._strategy_losses, create: `self._strategy_losses[strategy_type] = deque(maxlen=100)`
   - Append trade result: `self._strategy_losses[strategy_type].append(trade_won)`
   - Count consecutive trailing losses (from end of deque backwards):
     ```python
     losses = 0
     for result in reversed(self._strategy_losses[strategy_type]):
         if not result:  # False = loss
             losses += 1
         else:
             break
     ```
   - If losses == 3 (FIRST time hitting 3, not repeated alerts):
     - Call `self.alert(AlertType.LOSS_STREAK, Severity.WARNING, f"Strategy {strategy_type}: 3 consecutive losses", {...})`
     - Include context dict: {"strategy": strategy_type, "loss_count": losses, "lookback_trades": len(self._strategy_losses[strategy_type])}
     - Return True (alert fired)
   - Otherwise return False
   - Never raise exception; log and continue

3. Add method `check_zero_opp_period(self, strategy_opportunities: dict[str, int]) -> None`:
   - Input: dict mapping strategy_type → count of opportunities detected in current scan (e.g. {"binary": 5, "cross": 0, "kalshi": 2})
   - For each strategy_type in input:
     - If count > 0: update `self._strategy_last_opp_time[strategy_type] = time.time()`; reset counter
     - If count == 0: check if current time - last_opp_time >= 1800 (30 min = 1800 seconds)
       - If yes: fire alert `self.alert(AlertType.ZERO_OPP, Severity.INFO, f"Strategy {strategy_type}: no opportunities for 30min", {...})`
   - Ensure each strategy only alerts once per 30-min window (rate limiting via 5-min AlertManager cooldown)

4. Add method `record_strategy_opportunity(self, strategy_type: str) -> None`:
   - Helper called from continuous.py after each scan
   - Updates `self._strategy_last_opp_time[strategy_type] = time.time()`
   - Called regardless of whether opportunities were detected (ensures tracking is initialized)

5. Log all alerts: `logger.warning("Alert: %s strategy %s", alert_type, strategy_type)`

| Boundary | Description |
|----------|-------------|
| Executor → AlertManager | Executor passes trade results; AlertManager trusts profitability flag is correct |
| Continuous → AlertManager | Continuous passes opportunity counts; AlertManager trusts counts are accurate |
| AlertManager → Notifier | Alerts fired via webhook; notifier auto-detects format (Slack/Discord) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Denial of Service | Alert spam from repeated loss streaks | mitigate | Use existing AlertManager rate limiting (5-min cooldown); log all alert fires; monitor alert frequency |
| T-06-11 | Information Disclosure | Alerts expose strategy names and loss details | mitigate | Strategy names are not secrets (e.g., "binary", "cross"); loss counts are internal metrics; acceptable for operator consumption |
| T-06-12 | Tampering | Manual loss streak injection by modifying _strategy_losses | mitigate | _strategy_losses is in-memory dict; no persistence; resets on bot restart; alerts are read-only (no operator input) |

</threat_model>
