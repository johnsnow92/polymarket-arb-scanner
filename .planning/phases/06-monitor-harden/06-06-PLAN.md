<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Health probe → Platform API | Sends auth credentials; trusts response is genuine (no man-in-the-middle) |
| Platform API → Health probe | Returns auth success/failure; health checker trusts response is accurate |
| Health checker → Alert manager | Fires alerts; alert manager delivers via webhook |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-19 | Denial of Service | Health checks called too frequently, blocking other operations | mitigate | Run health checks in separate background task (30-min interval); main scan loop unaffected |
| T-06-20 | Information Disclosure | Credential status exposed via alert webhook | mitigate | Credential health status is acceptable for operator consumption (no API keys in alerts) |
| T-06-21 | Spoofing | False credential success due to mocked response | mitigate | Use cheap endpoints (`/markets?limit=1`) that require valid auth; mock server can't spoof all platforms |
| T-06-22 | Elevation of Privilege | Health checker has access to modify credentials | mitigate | Health checker only reads auth status; never modifies or stores credentials |

</threat_model>
