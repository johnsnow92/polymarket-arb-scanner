**B. Calibration logging (modify `_revalidate` dispatcher):**

In the `_revalidate()` method (line ~309), add timing and structured logging around EVERY revalidation decision. After the dispatcher calls the specific `_revalidate_*` method:

```python
def _revalidate(self, opportunity: dict, price_cache: dict | None = None) -> bool:
    start_ms = int(time.time() * 1000)
    opp_type = opportunity.get("type", "")
    layer = opportunity.get("_layer")
    if layer is None:
        layer = get_layer(opp_type)
    
    # Get scan-time ROI for logging
    original_profit = float(opportunity.get("net_profit", opportunity.get("_net_profit", 0)))
    total_cost_raw = opportunity.get("total_cost", "$0")
    total_cost = float(str(total_cost_raw).replace("$", "")) if total_cost_raw else 0
    scan_roi = original_profit / total_cost if total_cost > 0 else 0
    
    # ... existing dispatcher logic (switch on opp_type) ...
    # Let result = True/False from the _revalidate_* call
    # Let reason = "passed" if result else rejection reason
    
    elapsed_ms = int(time.time() * 1000) - start_ms
    floor = REVAL_FLOORS.get(layer, 0.02)
    
    # Structured calibration log (per D-01)
    logger.info(
        "REVAL|layer=%d|type=%s|scan_roi=%.4f|reval_roi=%.4f|delta=%.4f|"
        "passed=%s|reason=%s|elapsed_ms=%d|floor=%.4f",
        layer, opp_type, scan_roi, reval_roi, scan_roi - reval_roi,
        result, reason, elapsed_ms, floor
    )
    
    return result
```

To capture `reval_roi` and `reason`, modify the `_revalidate_*` methods to return a tuple `(passed: bool, reval_profit: float, reason: str)` instead of just `bool`. Alternatively, add the structured log inside each `_revalidate_*` method before returning. Choose the approach that minimizes changes — if modifying return types would cascade too much, add the log line inside `_revalidate()` using available data.

The log format uses pipe-delimited key=value pairs so a post-processing script can parse Railway logs to compute per-layer 80th-percentile drift.
