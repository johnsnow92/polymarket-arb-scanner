<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Polygonscan API → whale transaction detection | Untrusted API responses; could include stale/replayed transactions or spam |
| Whale wallet addresses → WHALE_WALLETS config | User-supplied wallet list could include invalid or phishing addresses |
| CLOB contract events → trade direction identification | Transaction calldata parsing is complex; incorrect parsing could mirror wrong direction |
| Market price data → position sizing | Stale Polymarket prices could cause over-sized positions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07 | Spoofing | Wallet address validation | mitigate | Validate WHALE_WALLETS addresses on startup: all must match pattern 0x[0-9a-fA-F]{40} (40 hex chars); reject invalid with ConfigError |
| T-09-08 | Replay | Stale whale transactions | mitigate | Check transaction timestamp in _refine_whale_copy_with_prices(); drop trades older than 30s (outside latency budget); log dropped transactions |
| T-09-09 | Information Disclosure | Polygonscan API key in logs | mitigate | Never log POLYGONSCAN_API_KEY; sanitize error messages to log only "rate_limit_error" or "timeout", not full API response |
| T-09-10 | Denial of Service | Rate limit exhaustion | mitigate | Implement exponential backoff via tenacity (already in PolygonscanClient); monitor request rate (requests per second); log 429s with wallet address; alert if sustained rate limits |
| T-09-11 | Tampering | Whale position limit bypass | mitigate | Increment _whale_copy_position_count on execute() success, decrement on trade close; enforce >= 5 check before _build_legs() |
| T-09-12 | Denial of Service | POLYMARKET_CLOB_ADDRESS typo | mitigate | Verify POLYMARKET_CLOB_ADDRESS == "0x4bfb41d5b3570defd03c39a9a4d8de6bd8b8982e" on startup (hardcoded, not configurable); test with live Polygonscan to confirm transactions match |

</threat_model>
