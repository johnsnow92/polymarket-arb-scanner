<objective>
Implement combinatorial logical arbitrage strategy that detects semantic inconsistencies across related Polymarket markets using config-driven rule sets.

**Purpose:** Enable profitable trades on logically-implied market relationships (e.g., "Bitcoin >$100k" implies "Bitcoin >$90k"). If the implied outcome is priced significantly lower than the implying outcome, buy the implied outcome and sell the implying outcome.

**Output:** New scan module (scans/logical_arb.py), fee calculator in fees.py, executor integration in executor.py, and comprehensive unit tests in tests/test_logical_arb.py.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LOGICAL_ARB_RULES config → scan logic | Untrusted user-supplied JSON rules could encode circular dependencies or malformed market IDs |
| Market price data → opportunity detection | Stale or spoofed prices from API could trigger false positive opportunities |
| Polymarket CLOB order book → revalidation | CLOB data is live but can be manipulated by large orders; Stage 2 refinement validates freshness |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | LOGICAL_ARB_RULES JSON | mitigate | Validate JSON schema at config load (config.py:validate_logical_arb_rules checks for if_yes/then_yes/relationship keys); reject malformed rules with ConfigError |
| T-09-02 | Denial of Service | LOGICAL_ARB_RULES cycles | mitigate | Detect rule cycles via directed graph traversal at startup; log warning if A→B→A cycle detected and disable rule pair |
| T-09-03 | Tampering | Stale CLOB prices in Stage 2 | mitigate | Compare Stage 1 estimated price vs Stage 2 CLOB ask price; drop if spread >30% (detects spoofed/stale order books) |
| T-09-04 | Denial of Service | Missing market IDs in rules | mitigate | Log missing markets at scan time, continue with available rules (graceful degradation) |
| T-09-05 | Information Disclosure | Polymarket API errors | mitigate | Never log full API response body (contains potentially sensitive data); log sanitized error messages with code only |
| T-09-06 | Repudiation | Executor fee mismatch | mitigate | net_profit_logical_arb() uses fixed POLYMARKET_TAKER_FEE from config (immutable); audit fees.py on every deployment |

</threat_model>
