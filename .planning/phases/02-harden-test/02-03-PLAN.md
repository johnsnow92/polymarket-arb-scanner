  <action>
    1. Create `tests/integration/test_strategies.py` with the following structure:

       - `import subprocess, sys, os, time` at the top
       - `sys.path.insert(0, os.path.join(os.path.dirname(__file__), "..", ".."))` for project root
       - A `PROJECT_ROOT` constant pointing to the repo root (2 levels up from tests/integration/)
       - A helper function `_run_scanner(mode: str, timeout: int = 120) -> tuple[int, str, str]`:
         ```python
         def _run_scanner(mode, timeout=120):
             result = subprocess.run(
                 [sys.executable, os.path.join(PROJECT_ROOT, "scanner.py"),
                  "--mode", mode, "--dry-run"],
                 capture_output=True, text=True, timeout=timeout,
                 cwd=PROJECT_ROOT,
             )
             return result.returncode, result.stdout, result.stderr
         ```

       - Credential check helpers:
         ```python
         def _has_polymarket_creds():
             return bool(os.getenv("POLYMARKET_PRIVATE_KEY"))

         def _has_kalshi_creds():
             return bool(os.getenv("KALSHI_API_KEY_ID"))

         def _has_betfair_creds():
             return all(os.getenv(k) for k in ("BETFAIR_APP_KEY", "BETFAIR_USERNAME", "BETFAIR_PASSWORD"))

         def _has_smarkets_creds():
             return bool(os.getenv("SMARKETS_API_KEY"))

         def _has_sxbet_creds():
             return bool(os.getenv("SXBET_API_KEY"))

         def _has_matchbook_creds():
             return all(os.getenv(k) for k in ("MATCHBOOK_USERNAME", "MATCHBOOK_PASSWORD"))

         def _has_gemini_creds():
             return all(os.getenv(k) for k in ("GEMINI_API_KEY", "GEMINI_API_SECRET"))

         def _has_ibkr_creds():
             return bool(os.getenv("IBKR_HOST"))
         ```

       - One test class `TestStrategyDryRun` with methods for each mode. Each test:
         a. Checks credentials via the appropriate helper. If missing, `pytest.skip("No <platform> credentials")`
         b. Calls `_run_scanner(mode)`
         c. Asserts `returncode == 0`
         d. Asserts `"Traceback" not in stderr`
         e. Records stdout snippet for evidence

       - Modes and their credential requirements:
         - `test_binary_dry_run` -- needs polymarket creds
         - `test_negrisk_dry_run` -- needs polymarket creds
         - `test_cross_dry_run` -- needs polymarket + kalshi creds
         - `test_kalshi_dry_run` -- needs kalshi creds
         - `test_cross_all_dry_run` -- needs polymarket + kalshi creds (minimum pair)
         - `test_spread_dry_run` -- needs polymarket + kalshi creds
         - `test_betfair_dry_run` -- needs betfair creds
         - `test_smarkets_dry_run` -- needs smarkets creds
         - `test_sxbet_dry_run` -- needs sxbet creds
         - `test_matchbook_dry_run` -- needs matchbook creds
         - `test_gemini_dry_run` -- needs gemini creds
         - `test_ibkr_dry_run` -- needs ibkr creds
         - `test_event_dry_run` -- no creds needed (Metaculus public API)
         - `test_triangular_dry_run` -- needs polymarket + kalshi creds
         - `test_multi_cross_dry_run` -- needs polymarket + kalshi creds
         - `test_stale_dry_run` -- needs polymarket creds. Note: one-shot stale mode is a no-op; test asserts exit 0 and does NOT assert on finding opportunities.
         - `test_resolution_dry_run` -- needs polymarket + kalshi creds
         - `test_convergence_dry_run` -- needs polymarket + kalshi creds
         - `test_mm_dry_run` -- needs polymarket creds

       - Each test method stores its result (pass/fail/skip, evidence snippet) on the class for report generation.

    2. Create `tests/integration/run_all.py` -- a standalone orchestrator:
       - Imports `test_strategies` module
       - Runs each `test_*_dry_run` function in a try/except
       - Collects results into a list of dicts: `{"mode": str, "status": "PASS"|"FAIL"|"SKIP", "evidence": str, "timestamp": str}`
       - **Also runs `verify_fees.py`** as a subprocess and captures its pass/fail:
         ```python
         # Run fee verification (HARDEN-02 evidence)
         fee_result = subprocess.run(
             [sys.executable, os.path.join(PROJECT_ROOT, "tests", "integration", "verify_fees.py")],
             capture_output=True, text=True, timeout=30,
             cwd=PROJECT_ROOT,
         )
         fee_status = "PASS" if fee_result.returncode == 0 else "FAIL"
         results.append({
             "mode": "fee-verification",
             "status": fee_status,
             "evidence": fee_result.stdout[-300:] if fee_result.stdout else fee_result.stderr[-300:],
             "timestamp": datetime.now().isoformat(),
         })
         ```
       - Generates `RESULTS.md` at `.planning/phases/02-harden-test/RESULTS.md` with format:
         ```markdown
         # Phase 2: Harden & Test -- Integration Test Results

         **Run date:** YYYY-MM-DD HH:MM:SS
         **Total:** X tested, Y passed, Z failed, W skipped

         ## Strategy Dry-Run Tests (HARDEN-01)

         | Strategy | Mode | Status | Evidence | Timestamp |
         |----------|------|--------|----------|-----------|
         | Binary arbs | binary | PASS | 3 candidates found | 2026-03-20 |
         ...

         ## Fee Verification (HARDEN-02)

         | Check | Status | Evidence | Timestamp |
         |-------|--------|----------|-----------|
         | All 8 platform fees match documented rates | PASS/FAIL | verify_fees.py output | 2026-03-20 |
         ```
       - Exits 0 if no FAIL results, exits 1 if any FAIL.
       - Runnable via `python tests/integration/run_all.py`
