   ```python
   def log_reward_metric(self, platform: str, market_key: str, order_id: str, event: str,
                        size: float, spread: float, resting_seconds: int) -> None:
       """Log Kalshi reward tracking metrics (no Polymarket public API).
       
       Args:
           platform: "polymarket" or "kalshi"
           market_key: Market identifier
           order_id: Exchange order ID
           event: "placed" or "cancelled"
           size: Order size in dollars
           spread: Bid-ask spread as decimal (0.05 = 5%)
           resting_seconds: Seconds order has been resting
       """
       with self._lock:
           cursor = self._conn.cursor()
           cursor.execute("""
               INSERT INTO reward_metrics (platform, market_key, order_id, event, size, spread, 
                                          resting_seconds, timestamp)
               VALUES (?, ?, ?, ?, ?, ?, ?, ?)
           """, (platform, market_key, order_id, event, size, spread, resting_seconds, int(time.time())))
           self._conn.commit()
   ```

3. Create reward_metrics table if it doesn't exist (in create_tables()):
   ```sql
   CREATE TABLE IF NOT EXISTS reward_metrics (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       platform TEXT NOT NULL,
       market_key TEXT NOT NULL,
       order_id TEXT,
       event TEXT NOT NULL,
       size REAL,
       spread REAL,
       resting_seconds INTEGER,
       timestamp INTEGER NOT NULL
   )
   ```

This enables local logging of Kalshi orders (since Kalshi has no public reward API) and persistence of reward metrics for audit trail.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Polymarket Markets API → RewardTracker | Untrusted reward metadata (min_size, max_spread, pool_size) could be malformed |
| Kalshi order data → KalshiRewardTracker | Order placement data from exchange is trusted (verified by nonce/auth) |
| Database → Reward metrics lookup | SQLite data is authoritative local source of truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Polymarket reward metadata validation | mitigate | Validate all reward fields before use: check min_size > 0, 0 < max_spread <= 0.50, pool_size >= 0; reject markets with invalid data (log warning, skip) |
| T-07-02 | Denial of Service | RewardTracker cache exhaustion | accept | Single dict per market; bounded by # of Polymarket markets (~5k); no unbounded growth |
| T-07-03 | Information Disclosure | Kalshi order logs in database | accept | Logs contain only order metrics (size, spread, time) — no PII, no credentials, no account balances |
| T-07-04 | Tampering | Database reward_metrics table poisoning | accept | Database is local SQLite with WAL mode; file-level permissions sufficient. No multi-user access. |

</threat_model>
