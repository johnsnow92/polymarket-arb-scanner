Structure:

1. scan_polymarket_rewards(markets: list[dict], reward_tracker, min_pool_usdc: float = 10.0, price_cache: dict | None = None) -> list[dict]:
   - Stage 1: Filter markets with active reward programs (check incentives field from Markets API)
   - Validate reward metadata: min_size > 0, 0 < max_spread <= 0.50, pool_size >= min_pool_usdc
   - Reject markets with zero/inactive incentives
   - For each reward-eligible market:
     * Get mid-price from Markets API or price_cache
     * Calculate optimal reward-aware bid/ask via reward_tracker.calculate_optimal_reward_spread()
     * Check midpoint range: if 0.10 <= mid <= 0.90, single-sided OK; else require both sides
   - Return opportunities with type='PolymarketRewards', _layer=3, market_key, optimal_bid, optimal_ask, reward_pool_usdc, min_size

2. _refine_rewards_with_clob(opportunities: list[dict], markets_by_key: dict, price_cache: dict | None = None) -> list[dict]:
   - Stage 2: Re-check optimal quotes against live CLOB order book
   - For each opportunity, fetch actual best bid/ask from CLOB
   - Verify optimal quotes are achievable (not crossing market)
   - Filter out opportunities where CLOB depth too shallow (<$5)
   - Keep all opportunities if CLOB unavailable (graceful degradation)

3. scan_kalshi_rewards(kalshi_client, reward_tracker, min_pool_usdc: float = 10.0) -> list[dict]:
   - Fetch active Kalshi markets via kalshi_client.list_markets()
   - For each market with sufficient volume (daily volume > $1000):

- Target spread = max_incentive_spread * 0.6 (higher score without competing too hard)
- Double-sided requirement outside [0.10, 0.90] midpoint range
- Minimum reward pool threshold: $10 (skip tiny pools)

```python
def net_profit_rewards(bid_price: float, ask_price: float, size: float = 1.0,
                      platform: str = "polymarket") -> dict:
    """Calculate net profit for reward resting orders.
    
    For rewards strategy, profit = mid-price capture (spread) + expected reward payout.
    This is simplified: we track the spread capture and reward yield separately in the database.
    For execution, we calculate the spread profit per fill.
    
    Args:
        bid_price: Resting bid price
        ask_price: Resting ask price
        size: Order size in dollars
        platform: "polymarket" or "kalshi"
    
    Returns:
        Dict with net_profit, spread, fees, net_roi
        For rewards: spread profit is primary; reward_yield tracked separately in DB
    """
    mid = (bid_price + ask_price) / 2
    spread = ask_price - bid_price
    
    # Platform fees (simplified for resting limit orders)
    if platform == "polymarket":
        # Polymarket maker fee for reward-qualified orders: 0% (if reward eligible)
        # But actual fee rate may apply; use 0.005 conservative estimate
        fee_rate = 0.005  # 0.5% effective (may be lower or zero for rewards)
    elif platform == "kalshi":
        # Kalshi maker fee: capped at $1.75 per side
        fee_rate = 0.005
    else:
        fee_rate = 0.01  # Default conservative fee
    
    fees = spread * size * fee_rate
    net_profit = spread * size - fees
    net_roi = (net_profit / (size)) * 100 if size > 0 else 0.0
    
    return {
        "net_profit": net_profit,
        "spread": spread,
        "fees": fees,
        "net_roi": net_roi,
        "bid": bid_price,
        "ask": ask_price,
    }
```
