## Root Cause: Production Execution Failure

The bot detects ~10,473 opportunities per 7-day window but executes 0. Two skip reasons:

1. **`stale_prices`** — Revalidation re-fetches prices and finds profit below threshold. Previously: any API failure also returned `False` (counted as stale). **Fix committed:** API failures now raise `_RevalidationAPIError`, accepted if ROI >= 2%.

2. **`no_legs`** — `_build_legs()` returns empty. Possible causes: price string parsing failure, platform not in `ENABLED_EXECUTION_PLATFORMS`.

**Local fixes committed (not yet deployed):**
- `REVALIDATION_MIN_FLOOR`: 0.003 → 0.001
- `WS_CACHE_MAX_AGE_REVALIDATION`: 5s → 15s
- API errors tolerated on high-ROI (>= 2%) opportunities
- Enhanced diagnostic logging in all revalidation methods
