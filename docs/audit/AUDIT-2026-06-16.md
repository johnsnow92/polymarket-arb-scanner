
# Codebase Audit Report — polymarket-arb-scanner
**Date:** 2026-06-16  
**Scope:** All Python source files (`.`, `scans/`, `scripts/`)  
**Dimensions:** Bugs, Security  
**Methodology:** Multi-agent fan-out (40 finder agents) → 3-lens adversarial verification → ranked synthesis  

---

## Executive Summary

The scanner has **42 confirmed HIGH-severity bugs** and **14 confirmed HIGH-severity security findings** across its ~40 production modules. The most critical bugs are a cluster of `NameError`/`ImportError` crashes in `continuous.py` that kill the process whenever any Layer-4 strategy flag is enabled, wrong profit-calculation formulas in `fees.py` that systematically overstate or understate earnings (directly affecting live trade decisions), and inverted arb-detection conditions in four scan modules that cause the scanner to skip real opportunities and keep unprofitable ones. On the security side, six SSRF vectors let an attacker who can set env vars redirect authenticated financial API calls to internal network hosts; the trading dashboard exposes a kill-switch and fund-transfer endpoint without authentication when `DASHBOARD_PASS` is unset; and the Polymarket private key is retained as a plain string throughout the process lifetime. No finding required destructive testing — all were identified by static analysis and confirmed by independent code review.

## Coverage Gaps Identified by Completeness Critic

- **SQLite thread-safety**: `TradeDB` uses a single connection with a `threading.Lock`, but `dashboard.py:730` accesses `db.conn` directly, bypassing the lock — a concurrent write from the scan thread and a dashboard read race on the SQLite connection object.
- **Session token lifecycle**: No platform API client (Betfair, Smarkets, Matchbook) implements token expiry tracking or automatic re-authentication — long-running sessions will eventually return 401s that are mis-handled as transient errors.
- **Dashboard rate limiting**: No rate limiting on the HTTP dashboard server; unauthenticated (or authenticated) endpoints can be flooded to exhaust the thread pool serving the dashboard.
- **`DATA_DIR` path traversal**: Multiple modules construct file paths as `os.path.join(DATA_DIR, filename)` without canonicalizing or restricting to a safe prefix; a `DATA_DIR=../../etc` env var writes/reads outside intended data directory.
