# Codebase Audit Report — arbgrid (polymarket-arb-scanner)

**Generated:** 2026-06-28
**Scope:** 229 Python files — orchestration shell, scans/ detection package, platform API clients, execution/persistence layer
**Methodology:** 3-lens adversarial verification on each finding; only findings that survived all three lenses are reported here.

---

## Executive Summary

**Verdict:** The codebase has serious, exploitable defects in its financial core. Several bugs directly threaten capital: a daily-loss-limit bypass via P&L mixing, a double-spend on fund transfers, deadlocks in capital-optimization and geographic-routing code, and systematically wrong position sizing. Two SSRF-class vulnerabilities allow authenticated traffic (including signed orders and private key-derived auth) to be redirected to attacker-controlled hosts via environment variable manipulation. The test suite has a pervasive pattern of vacuous and tautological assertions that provide false confidence across multiple subsystems. None of these findings are theoretical edge cases — each maps to a direct code path that runs in normal operation.

## Confirmed Findings — HIGH Severity

### H-01 — Daily P&L mixes realized and unrealized, bypassing the loss limit
**File:** `db.py:252` | **Dimension:** Bugs

`get_daily_pnl()` returns `realized + row2["total"]` where `row2` is open positions' `expected_pnl` (unrealized). `RiskManager.check()` calls this to enforce the daily loss limit. Example: −$200 realized + $300 unrealized = +$100 reported — no limit triggered. **Fix:** Return only `SUM(actual_pnl)` from settled positions. Expose a separate `get_unrealized_pnl()` for display only.

---

### H-02 — Fund transfer idempotency gate does not prevent double-spend
**Files:** `tests/test_treasury.py:176` (documents bug), `treasury.py` (root cause) | **Dimension:** Bugs / Security

The test comment at line 186 explicitly states "the live withdraw is called twice". `db.log_transfer()` catches the UNIQUE-key collision and silently returns the existing row's id, but `execute_transfer()` always falls through to the live dispatch. No assertion on `gemini.withdraw_usdc.call_count`. A retry or race condition on the same idempotency key triggers two on-chain USDC withdrawals. **Fix:** Make `log_transfer` return `(id, is_new: bool)`. In `execute_transfer`, early-return when `is_new` is False, before dispatching to the corridor.

### H-03 — Deadlock in `TaxOptimizer.get_harvest_candidates` (non-reentrant lock)
**File:** `capital_optimizer.py:314` | **Dimension:** Bugs

`get_harvest_candidates` holds `self._lock` (line 302), then calls `self.is_long_term()` (line 314) which tries `with self._lock:` again. `threading.Lock` is not reentrant — guaranteed deadlock whenever any position qualifies. **Fix:** Change to `threading.RLock()`, or inline the `is_long_term` check without re-acquiring.

### H-04 — Kelly position sizer misinterprets percentage ROI strings, inflating size up to 100x
**File:** `position_sizer.py:354` | **Dimension:** Bugs

`net_roi` is formatted as `f"{profit/cost*100:.2f}%"` throughout the codebase (e.g. `"0.10%"`, `"0.50%"`). After stripping `%`, these are ≤ 1, so `abs(val) > 1` is false and the code returns 0.10 for a 0.1% ROI instead of 0.001 — a 100x inflation. **Fix:** Always divide by 100 after stripping `%`. Remove the heuristic.

### H-05 — Deadlock in `RegionRouter.should_route_externally` (non-reentrant lock)

Permanent deadlock whenever `GEOGRAPHIC_LATENCY_ENABLED=True` with region data present. **Fix:** Use `threading.RLock()`, or inline the best-region logic inside the existing lock block.

### H-06 — CRITICAL credential alert never fires on exception path; fires only once on unhealthy path
**File:** `credential_health.py:108, :94` | **Dimension:** Bugs

Exception path increments `_consecutive_failures` but never checks the count or calls `_fire_credential_alert`. The healthy=False path uses `== 3` (not `>= 3`), so after the 3rd failure the alert is permanently suppressed. Network errors (most common failure mode) never trigger the CRITICAL alert. **Fix:** Add `fail_count >= 3` check in the exception handler; change `== 3` to `>= 3` in both locations.

### H-13 — Stale price scan computes fees as % of profit, not % of trade value — understates fees ~10x

### H-14 — Betfair/exchange settlement always uses arb payout formula, misreporting directional losses
**File:** `continuous.py:383` | **Dimension:** Bugs

`_calc_realized_pnl(winning_side=None)` applies `payout = 1.0 - total_fill_cost`. A losing $0.80 directional bet is recorded as +$0.20 profit. Inflates the P&L database, undermines the daily loss limit, corrupts performance reporting for all directional strategies on exchange platforms. **Fix:** Query the winning runner from the platform API and pass `winning_side` explicitly.

### H-15 — `_is_settled` misclassifies exchange market winners as 'no' when name lacks 'yes'

`"yes" if "yes" in str(winner).lower() else "no"` — exchange outcomes like `"Team A"` or `"Over"` are always classified as `"no"`. The strategy buys the wrong side, guaranteeing a loss. **Fix:** Normalize winner to `'yes'`/`'no'` at the platform client level, or compare against a stored `yes_runner_id`.

### H-16 — Cross-platform concurrent execution does not abort unfired legs after a leg failure
**File:** `executor.py:2573` | **Dimension:** Bugs

All legs are submitted to `ThreadPoolExecutor` simultaneously. On failure, no `Future.cancel()` is called on not-yet-started legs. For a 3-leg triangular arb, legs 0 and 1 can both fill while leg 2 fails, requiring simultaneous hedging of two filled positions. **Fix:** After detecting a failure, call `future.cancel()` on not-yet-dispatched futures and set a cancellation flag.

### H-17 — SSRF: WebSocket proxy URLs bypass the `url_guard.py` SSRF guard
**File:** `ws_feeds.py:93` | **Dimension:** Security

`POLYMARKET_PROXY_URL` and `KALSHI_PROXY_URL` are read from env vars and passed directly to `Proxy.from_url()` without validation. An attacker controlling a Railway env var can tunnel authenticated WebSocket traffic through an internal address (e.g. `socks5://169.254.169.254:1080`). `assert_public_url` is already applied to REST endpoints but not here. **Fix:** Apply `assert_public_url` at `FeedManager` construction time.

### H-18 — SSRF: all six REST proxy env vars forwarded to `requests.Session.proxies` without validation
**Files:** `polymarket_api.py:34`, `betfair_api.py:48`, `smarkets_api.py:47`, `matchbook_api.py:44`, `gemini_api.py:80`, `kalshi_api.py:91` | **Dimension:** Security

RSA-signed Kalshi headers, HMAC-SHA384 Gemini signatures, and Betfair session tokens can be redirected through attacker-controlled hosts. **Fix:** Validate each proxy URL with `assert_public_url` before assigning to `session.proxies`.

**File:** `fees.py:34` | `prob_a * case_b_fees + (1.0 - prob_a) * case_a_fees` — weights transposed.
