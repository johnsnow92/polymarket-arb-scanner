# Arbgrid Codebase Audit Report
**Generated:** 2026-06-29
**Scope:** Python, ~229 files — Arbgrid prediction-market arbitrage bot
**Methodology:** 153 confirmed findings (each survived a 3-lens adversarial vote), 0 unverified

## Executive Summary

**Verdict: Do not trade live until the critical and high findings are resolved.**

The scanner contains 10 confirmed critical/high bugs — several of which are silent no-ops that make core safety controls (Stage-2 CLOB refinement, WS-triggered Kalshi execution, crash-recovery hedging, tax-optimizer locking) completely inoperative. A further 93 medium findings span incorrect financial formulas, security vulnerabilities, and vacuous tests. 50 low-severity issues add up to substantial technical debt.

| Severity | Count |
|----------|-------|
| Critical / High | 10 |
| Medium | 93 |
| Low | 50 |
| **Total confirmed** | **153** |
| Unverified | 0 |

The three most dangerous issue classes are:
1. **Silent execution bypasses** — Stage-2 CLOB refinement for scan_cross_all is a complete no-op; Kalshi WS-triggered execution is silently suppressed; the tax-optimizer deadlocks.
2. **Wrong financial math** — fee EV weights are swapped; Sharpe computes annualized volatility instead of Sharpe; realized P&L hardcoded to $1 payout regardless of position size; stale/convergence/time-decay fee formulas apply fees to profit instead of capital.
3. **Security gaps** — ~/.claude/.env secrets injected into the trading process; four proxy URLs unguarded against SSRF; Supabase service-role key used where anon key suffices; dashboard unauthenticated GET endpoints in production.

### CRITICAL / HIGH (10)

**H-01 — Stage-2 CLOB refinement for scan_cross_all is a complete no-op**
`scans/cross.py:537`
Evidence: `_refine_cross_all_with_clob(opportunities, min_profit, price_cache=price_cache)` — return value discarded.
_refine_cross_all_with_clob builds a refined_out list internally but never appends to it and never returns it (returns None). The caller at line 537 also discards whatever it returns. Every Stage-1 over-estimated opportunity flows to the executor and is rejected only at order-placement time or executed at a loss.
Fix: Rebuild _refine_cross_all_with_clob to append surviving opportunities to refined_out and return it. Change the call site to `opportunities = _refine_cross_all_with_clob(...)`.

**H-02 — _refine_cross_all_with_clob mutates in-place but never removes failing entries**
`scans/cross.py:593`
Evidence: `refined_out = []` (dead variable); `o["net_profit"] = best_r["net_profit"]` (mutates in-place).
Opportunities that fail CLOB refinement remain in the opportunities list unchanged. Together with H-01, every cross-all opportunity propagates to execution without accurate ask-price validation.
Fix: Append surviving opportunities to refined_out, return it, use it at the call site.

**H-03 — Deadlock: TaxOptimizer.get_harvest_candidates calls is_long_term while holding non-reentrant threading.Lock**
`capital_optimizer.py:302`
Evidence: `with self._lock: ... if self.is_long_term(position_id) ...` and `is_long_term` also calls `with self._lock:`.
Any thread calling get_harvest_candidates with at least one loss-exceeding position hangs permanently.
Fix: Change `self._lock = threading.Lock()` to `threading.RLock()`.

**H-04 — WS-triggered Kalshi execution silently skipped due to wrong price cache key**
`continuous.py:880`
Evidence: `new_price = cached.get("price")` — Kalshi WS messages use yes_price/yes, not price.
new_price is always None for Kalshi updates; `if new_price is None: continue` fires and suppresses all Kalshi WS-triggered execution.
Fix: `new_price = cached.get("price") or cached.get("yes") or cached.get("yes_price")`

**H-05 — _convert_orphans_to_partial_fills passes order_id as token_id — hedges fail to locate market after crash**
`recovery.py:270`
Evidence: `token_id=trade.get("order_id", "")`
Every hedge generated from an orphaned trade fails to find the market, leaving positions unhedged after a crash.
Fix: `token_id=trade.get("token_id") or trade.get("market", "")`

**H-06 — FINNHUB_API_KEY referenced at runtime but never imported in continuous.py — NameError kills news-snipe**
`continuous.py:1543`
Evidence: `if not FINNHUB_API_KEY:` — name never imported into the module.
NameError caught by outer except, news-snipe permanently unavailable in continuous mode.
Fix: Add FINNHUB_API_KEY to config imports in continuous.py.

**H-07 — rewards.py assigns raw outcomePrices[0] string as mid_price without float() conversion**
`scans/rewards.py:231`
Evidence: `yes_price = prices[0]; mid_price = yes_price` — Polymarket API returns strings.
`mid_price - half_spread` raises TypeError; scan crashes for any market without a WS-cached price.
Fix: `yes_price = float(prices[0])` with bounds check `0.0 < yes_price < 1.0`.

**H-08 — Treasury idempotency test misses double-spend: withdraw_usdc called twice, call_count never asserted**
`tests/test_treasury.py:186`
Evidence: Inline comment admits "the live withdraw is called twice"; `gemini.withdraw_usdc.assert_called_once()` absent.
Two on-chain USDC withdrawals for one intended transfer in production.
Fix: Add `gemini.withdraw_usdc.assert_called_once()`. In production, have log_transfer return (id, is_new) and return early when is_new=False.

**H-09 — supabase_sync.py: int() cast on timestamp crashes on None or non-numeric strings**
`supabase_sync.py:49`
Evidence: `ts = int(row.get('timestamp', 0))` — int(None) raises TypeError.
A nullable column aborts the entire sync_reward_metrics call.
Fix: `ts = int(float(row.get('timestamp') or 0))` inside try/except.

**H-10 — Sharpe ratio formula computes annualized volatility instead of Sharpe ratio**
`scripts/analytics.py:106`
Evidence: `stddev * SQRT(252)` — should be `(AVG(net_profit) / NULLIF(stddev, 0)) * SQRT(252)`.
All strategy rankings and comparisons to Sharpe thresholds (>1 = good) are meaningless.
Fix: `(AVG(net_profit) / NULLIF(STDDEV_SAMP(net_profit), 0)) * SQRT(252) as annual_sharpe`.
