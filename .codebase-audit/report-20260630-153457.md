## Executive Summary

The codebase carries two runtime deadlocks that permanently hang specific scan paths under concurrent load, a cross-platform scan bug that corrupts every CLOB-refined triangular opportunity and can send live orders at wrong prices, a treasury idempotency gap that allows a double USDC withdrawal, and an unvalidated fund-transfer destination address that can permanently lose funds to a wrong Ethereum address. Security findings include five proxy SSRF holes, three validation gaps on Sentry/Supabase URLs, and a plaintext credential in the process table.

#### C-1 — Deadlock in `TaxOptimizer.get_harvest_candidates` calling `is_long_term` while holding a non-reentrant lock

**File:** `capital_optimizer.py` **Line:** 314
**Dimension:** bugs

```python
# __init__: self._lock = threading.Lock()  (line 208)
with self._lock:          # line 302 — acquired
    rate = (TAX_LONG_TERM_RATE
            if self.is_long_term(position_id)   # line 314 — re-acquires self._lock
            else TAX_SHORT_TERM_RATE)
```

`threading.Lock` is not reentrant. Every call to `get_harvest_candidates` with any position in `self._cost_basis` deadlocks permanently. Tax-loss harvesting is silently dead.

**Fix:** Change `self._lock = threading.Lock()` to `self._lock = threading.RLock()`, or inline the `is_long_term` logic inside `get_harvest_candidates` without re-acquiring the lock.

#### C-2 — Deadlock in `should_route_externally`: calls `get_best_region` while holding `self._lock`

**File:** `latency_monitor.py` **Line:** 372
**Dimension:** bugs

with self._lock:                                    # line 366 — acquired
