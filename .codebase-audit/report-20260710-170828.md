## Executive Summary

The overall verdict is **not production-ready without targeted remediation**: four critical deadlocks and a price-calculation inversion can halt execution or systematically mis-size positions; a cluster of high-severity scan logic bugs causes false-positive arbitrage signals reaching the executor on every cycle; and three SSRF vulnerabilities exist in the platform API and data-sync layers.

#### C-1. Deadlock in `TaxOptimizer.get_harvest_candidates` — non-reentrant lock acquired twice

**File:** `capital_optimizer.py` — lines 302, 314, 256
**Dimension:** bugs

**Evidence:**
```
# Line 208: self._lock = threading.Lock()  (NOT RLock)
# get_harvest_candidates (line 302):
with self._lock:
    if pnl_pct < -threshold:
        rate = (TAX_LONG_TERM_RATE if self.is_long_term(position_id) ...)
# is_long_term (line 256):
def is_long_term(self, position_id: str) -> bool:
    with self._lock:   # tries to re-acquire same Lock -> DEADLOCK
```

**Why it matters:** Any call to `get_harvest_candidates` when `TAX_AWARE_ENABLED=true` deadlocks the calling thread permanently, silently halting the bot.

**Fix:** Change `self._lock = threading.Lock()` to `self._lock = threading.RLock()`, or inline the `is_long_term` logic inside the already-held lock without re-acquiring.

#### C-2. Deadlock in `RegionRouter.should_route_externally()` — same non-reentrant lock acquired twice

**File:** `latency_monitor.py` — lines 366, 372, 344
**Dimension:** bugs

**Evidence:**
```
def should_route_externally(self, platform):
    with self._lock:           # line 366 — acquires plain Lock
        best_region = self.get_best_region(platform)  # line 372
def get_best_region(self, platform):
```

**Why it matters:** With `GEOGRAPHIC_LATENCY_ENABLED=True`, every call to `should_route_externally()` blocks forever, making latency-optimised routing permanently inoperable.

**Fix:** Inline the `min()` computation from `get_best_region` directly inside the already-locked block in `should_route_externally`, eliminating the nested call.

#### C-4. `_select_fees` has inverted probability weights, systematically mispricing EV-model fees

**File:** `fees.py` — line 34
**Dimension:** bugs

**Evidence:**
```python
def _select_fees(case_a_fees, case_b_fees, price_a):
    if FEE_MODEL == 'expected_value':
        prob_a = max(0.0, min(1.0, price_a))
        return prob_a * case_b_fees + (1.0 - prob_a) * case_a_fees  # INVERTED
```

**Why it matters:** When `price_a=0.90`, the formula applies 90% weight to `case_b_fees` (NO-wins scenario) and only 10% to `case_a_fees` (YES-wins). Affects `net_profit_cross_betfair`, `net_profit_cross_smarkets`, `net_profit_cross_generic`, and `net_profit_triangular` — systematically inflating or deflating net profits whenever `FEE_MODEL=expected_value`.

**Fix:** `return prob_a * case_a_fees + (1.0 - prob_a) * case_b_fees`

#### H-3. CLOB key/market mapping inverted for ~50% of correlated pairs in Stage 2

**File:** `scans/correlated.py` — line 365
