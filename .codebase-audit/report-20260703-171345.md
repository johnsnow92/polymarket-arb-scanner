## CRITICAL

### C-01 — scan_multi_cross calls wrong function with wrong signature, crashing every multi-cross scan
**File:** `scans/multi_cross.py:254`

```python
elif kalshi_client:
    from scans.helpers import _parallel_fetch_kalshi
    kalshi_events, kalshi_by_event, kalshi_event_titles = _parallel_fetch_kalshi(kalshi_client)
```

`_parallel_fetch_kalshi` requires two arguments (`kalshi_client, tickers: list[str]`) and returns a plain `dict`, not a 3-tuple. This call site passes one argument (TypeError) and tries to unpack the result as a 3-tuple (ValueError). Every call to `scan_multi_cross` without pre-fetched Kalshi data crashes immediately.

**Fix:** `from scans.kalshi import _fetch_kalshi_data; kalshi_events, kalshi_by_event, kalshi_event_titles = _fetch_kalshi_data(kalshi_client)`

### C-02 — Deadlock in TaxOptimizer.get_harvest_candidates via non-reentrant lock
**File:** `capital_optimizer.py:302`

```python
with self._lock:                          # threading.Lock — non-reentrant
    ...
    if self.is_long_term(position_id):    # calls self._lock.acquire() → blocks forever
```

`is_long_term` (line 256) also calls `with self._lock:`. A non-reentrant lock cannot be acquired twice by the same thread. The second acquisition blocks forever, hanging the thread with the lock held and preventing any other thread from progressing.

**Fix:** Change `self._lock = threading.Lock()` to `self._lock = threading.RLock()`, or extract an `_is_long_term_unlocked()` helper that reads `self._cost_basis` without acquiring the lock.

## HIGH

### H-01 — Realized P&L uses price × size instead of size alone
**File:** `continuous.py:254`

```python
total_fill_cost = sum((t.get("fill_price") or t["price"]) * t["size"] for t in trades)
return 1.0 - total_fill_cost   # e.g. 1.0 - 19.4 = -18.4
```

`size` is in dollar terms. Multiplying by a per-contract fill price (0–1) gives a meaningless product. The arb path returns large negative P&L values, corrupting the positions table, the daily-loss check, and all downstream reporting.

**Fix:** `total_fill_cost = sum(t["size"] for t in trades)`

### H-14 — Sentry initialized without credential scrubbing — private keys sent to third-party error reporter
**File:** `sentry_init.py:15`
