## Executive Summary

**Verdict:** Significant defects block safe live trading. Two confirmed bugs corrupt P&L accounting and silently pass losing trades to the executor; three security issues (SSRF vectors) enable credential exfiltration via environment variables; and a 50-finding test-quality cluster means a large fraction of the test suite provides no actual regression protection.

**Top 5 themes by impact:**
1. **P&L / trade-decision arithmetic is wrong in multiple scan paths** — CLOB refinement routines produce false-positive or false-negative signals that reach the executor unfiltered.
2. **Security: three independent SSRF vectors** can redirect authenticated HTTPS traffic (SUPABASE_URL, SXBET_API_BASE_URL, Betfair stream host), plus a Telegram token leaks to logs.
3. **Threading / concurrency correctness** — a permanent deadlock in the latency router, two race conditions on shared data structures, and async coroutines driven sequentially.
4. **Core analytics are wrong** — the Sharpe ratio column computes annualised volatility (missing the mean-return numerator) and max-drawdown is the equity range, not the peak-to-trough decline.
5. **Test suite provides false confidence** — ~35 tests have assertions that are tautologically true, vacuously guarded, or check the wrong value; production regressions in money-moving paths pass silently.

#### C-1 — Permanent deadlock in RegionRouter.should_route_externally
**File:** `latency_monitor.py:366`

```python
def should_route_externally(self, platform):
    with self._lock:          # acquires threading.Lock (non-reentrant)
        best_region = self.get_best_region(platform)   # line 372

def get_best_region(self, platform):
    if not GEOGRAPHIC_LATENCY_ENABLED:
        return self.local_region   # NOT taken when flag is True
    with self._lock:               # tries to re-acquire same Lock -> blocks forever
```

When `GEOGRAPHIC_LATENCY_ENABLED=True` (the production flag), every call to `should_route_externally` hangs the calling thread permanently. Any continuous-mode code path that invokes region-aware routing is permanently stalled.

**Fix:** Extract a private `_get_best_region_locked()` helper that assumes the lock is already held and call that from inside `should_route_externally`. Alternatively replace `threading.Lock` with `threading.RLock`.

### HIGH (17)

Findings at this severity are incorrect financial calculations reaching the executor, security issues enabling credential exfiltration, or tests that validate incorrect production behaviour.

#### H-1 — _calc_realized_pnl multiplies fill-price x size — wrong units, wrong payout
**File:** `continuous.py:254`

`size` is a dollar amount (e.g. $5); `fill_price` is a per-contract price in [0,1]. For a $5 trade at 0.48 fill, `total_fill_cost = 2.4` instead of $5. The function then returns `1.0 - 2.4 = -$1.4` when the actual profit is ~$0.30. This corrupts `realized_pnl` in the positions table, making the daily loss limit gate unreliable.

**Fix:** `total_fill_cost = sum(t["size"] for t in trades)` — the USDC actually spent per leg.

---

#### H-2 — evaluate() stores swapped _kalshi_yes/_kalshi_no on inverted pairs
**File:** `cross_pair_index.py:280`

After the `if pair.inverted: k_yes, k_no = k_no, k_yes` swap (correct for profit calc), the output dict stores post-swap values as `_kalshi_yes`/`_kalshi_no`. The executor reads these to decide which Kalshi side to buy; on every inverted pair it buys the wrong side.

**Fix:** Snapshot `k_yes_orig, k_no_orig = k_yes, k_no` before the swap; use those in the output dict.

#### H-3 — check_all_platforms drives 8 async health checks sequentially, not concurrently
**File:** `credential_health.py:75`

Calling `async def` without `await` or `asyncio.create_task` returns a coroutine that has not started. The subsequent loop drives each coroutine one-at-a-time. With 8 platforms x 10 s timeout, worst-case wall time exceeds 80 s.

**Fix:** `results = await asyncio.gather(*[self._check_platform_health(p) for p in self.clients], return_exceptions=True)`

#### H-4 — Kalshi WebSocket handler stores YES bid price as YES ask price
**File:** `ws_feeds.py:435`

`kalshi_api.py:462-464` documents that Kalshi orderbooks contain bids only and `yes_ask = 1.0 - best_no_bid`. The WS handler stores the YES bid ladder directly as the ask. All WS-triggered scan runs see wrong ask prices, generating false arb signals.

**Fix:** `yes_ask = 1.0 - (no_ladder[0][0] / 100.0)` — mirror the computation in `kalshi_api.py`.

#### H-5 — _refine_cross_all_with_clob builds a refined_out list but never populates or returns it
**File:** `scans/cross.py:621`

`refined_out = []` is declared but never used. Opportunities failing the CLOB revalidation check are skipped via `continue` but remain in the parent `opportunities` list with stale mid-price `net_profit` values. Every stage-1 false positive survives into the final output.

**Fix:** Build an explicit `refined` list (as `_refine_cross_with_clob` does), append only passing candidates, and return it. The caller must assign the return value.

#### H-6 — _refine_triangular_with_clob always uses other_price = 0
**File:** `scans/triangular.py:414`

`scan_triangular` builds opp dicts without `_price_a`/`_price_b`. The refiner reads `o.get("_price_b", 0)` — always 0. `net_profit_triangular(pm_ask, 0, ...)` returns approximately `1.0 - pm_ask` (~55%) for every candidate, guaranteeing every post-CLOB triangular candidate passes and appears extremely profitable.

**Fix:** In `scan_triangular`, add `_price_a`, `_price_b`, `_side_a`, `_side_b` to the opp dict.

#### H-7 — guaranteed_gain in time-decay scan uses a fixed threshold, not the actual entry price
**File:** `scans/time_decay.py:81`

`guaranteed_gain = buy_below_price - target_price` uses the constant threshold (0.95) rather than `current_price`. When `consensus_prob > buy_below_price`, `guaranteed_gain` is negative but the opportunity is still emitted with no guard.

**Fix:** `guaranteed_gain = target_price - current_price`. Add `if guaranteed_gain <= 0: continue`.

#### H-8 — _refine_imbalance_with_clob passes sign-reversed signals through
**File:** `scans/imbalance.py:82`

`if abs(current_ratio) < collapse_threshold: continue` compares only magnitudes. A fully-reversed signal (original +0.50, live -0.40) passes because `abs(-0.40) = 0.40 > 0.35`. BUY opportunities with a flipped-to-bearish signal are forwarded to execution.

**Fix:** `if abs(current_ratio) < collapse_threshold or (original_ratio * current_ratio < 0): continue`

#### H-9 — outcomePrices strings not cast to float before arithmetic
**File:** `scans/rewards.py:231`

The Gamma API returns `outcomePrices` as `["0.63", "0.37"]` (strings). `mid_price = prices[0]` assigns a `str`; the first arithmetic in `_calculate_optimal_quotes` raises `TypeError`. Any market hitting the `outcomePrices` fallback path aborts processing.

**Fix:** `yes_price = float(prices[0])` with a `try/except ValueError` guard.

#### H-10 — Sharpe ratio SQL computes annualised volatility, not Sharpe ratio
**File:** `scripts/analytics.py:106`

```sql
THEN stddev * SQRT(252)   -- missing: avg_pnl / stddev in the numerator
```

`avg_pnl` is computed in the inner CTE but never used. The `annual_sharpe` column is annualised volatility, not a risk-adjusted return. It has wrong units, wrong scale, and opposite ranking behaviour.

**Fix:** `CASE WHEN trade_count >= 20 AND stddev > 0 THEN (avg_pnl / stddev) * SQRT(252) ELSE NULL END`

**Fix:** Update both the production SQL and the test to use the correct formula with a non-degenerate test input.

#### H-14 — SSRF via unvalidated SUPABASE_URL — service key exfiltration
**File:** `supabase_sync.py:76`

`url_guard.py` protects `POLYGON_RPC_URL`, `GEMINI_BASE_URL`, and `WEBHOOK_URL` but not `SUPABASE_URL`. The supabase-py client sends `SUPABASE_SERVICE_KEY` (which bypasses row-level security) as a Bearer token in every request. A redirected URL exfiltrates the high-privilege key.

**Fix:** `assert_public_url(url, env_name='SUPABASE_URL', allow_http=False)` before `create_client`.

#### H-15 — Betfair stream host configurable via env var without SSRF guard — session token exfiltrated
**File:** `ws_feeds.py:299`

`BETFAIR_STREAM_HOST` is read from env with no `assert_public_url` check. `BetfairFeed.connect()` sends the Betfair SSO session token as a plaintext JSON field in the first message after the TLS handshake — visible to any attacker-controlled host with a valid certificate.

#### H-16 — Cache eviction in record_resolution runs outside the lock
**File:** `calibration_tracker.py:104`

#### Logic / Calculation Bugs (13)

**M-1 — Reversed probability weights in _select_fees (EV fee model)**
`fees.py:34` — `prob_a * case_b_fees + (1-prob_a) * case_a_fees` has weights swapped. Affects `net_profit_cross_betfair`, `net_profit_cross_smarkets`, `net_profit_cross_generic`, and `net_profit_triangular`.
Fix: swap the terms to `prob_a * case_a_fees + (1-prob_a) * case_b_fees`.

**M-2 — Balance gate checks per-unit arb cost, not the actual dollar trade size**
`risk_manager.py:75` — `total_cost` is the combined ask price of one contract (e.g. 0.97). `trade_cost = min(max_trade_size, 0.97) = 0.97`. A $0.97 balance passes even though the trade will attempt to spend $25+.
Fix: compute the actual planned trade size using the same logic as `calculate_dynamic_size`.

**M-3 — classify_confidence receives an inflated combined score (can exceed 100)**
`matcher.py:174` — `best_score` = raw fuzzy + entity bonus (up to 15) + category bonus (up to 5) = up to 120. Thresholds designed for 0-100. A market with raw similarity 72 can be classified HIGH.
Fix: pass `best_raw_score` (before bonuses) to `classify_confidence`.

**M-4 — event_monitor.find_divergences includes platform price in the consensus it should exclude**
`event_monitor.py:271` — `add_signal(market_key, platform_name, platform_price)` is called before `get_consensus()`. The consensus probability pulls toward the platform's own price, shrinking measured divergences.
Fix: remove the `add_signal` call for the platform's own price before computing consensus.

**M-5 — _extract_net_roi returns raw percentage value for sub-1% ROI strings**
`position_sizer.py:355` — For `"0.50%"`, `val=0.50` does not trigger `abs(val) > 1`, returning 0.50 (50%) instead of 0.005 (0.5%). Kelly sizing uses a 100x inflated `net_roi`.
Fix: detect the `%` suffix and always divide by 100 when present.

**M-6 — Price filter in build_bucketed_series lets negative and >1 prices through**
`correlation_tracker.py:108` — Outer `if price <= 0 or price >= 1:` is entered for all out-of-range prices, but inner `if price == 0 or price == 1: continue` only fires for exact endpoints. Values like `-0.5` pass through into Pearson computation.
Fix: `if not (0 < price < 1): continue`

**M-7 — CRITICAL credential alert fires only when fail_count == 3; persistent failures beyond 3 are silent**
`credential_health.py:94` — `== 3` fires once; counts 4, 5, 6… match neither branch and produce no output.
Fix: `if fail_count >= 3:`

**M-8 — max_drawdown SQL computes equity range, not peak-to-trough decline**
`scripts/analytics.py:109` — `max_cumulative_pnl - min_cumulative_pnl` is the total range. A strategy with PnL `[10, 20, 30, 29, 59]` reports drawdown 49; true max drawdown is 1.
Fix: use a running-peak window function and aggregate `MAX(running_peak - cumulative_pnl)`.

**M-9 — max_drawdown test asserts wrong expected value**
`tests/test_analytics.py:167` — Comment says "0.08 - (-0.07) = 0.15"; this uses the final value (after the trough) not the pre-trough peak. Both production SQL and test agree on the inflated formula.

**M-10 — cross_category.days_remaining documented as affecting probability but never referenced**
`scans/cross_category.py:189` — All five return paths ignore `days_remaining`. A market expiring tomorrow and one expiring in a year produce identical implied probabilities.
Fix: implement time scaling or remove the parameter.

**M-11 — expert_divergence.min_forecasters is loaded but never enforced**
`scans/expert_divergence.py:63` — The value is assigned but never compared against `expert_forecast.get('num_sources', 1)`. Single-forecaster signals pass unchecked.
Fix: `if expert_forecast.get('num_sources', 1) < min_forecasters: continue`

**M-12 — SQLite connections in CalibrationTracker not closed on exception**
`calibration_tracker.py:94` — Five methods call `conn.close()` at end of block; any exception bypasses it.
Fix: `with sqlite3.connect(...) as conn:` or `try/finally`.

**M-13 — CalibrationTracker in-memory cache modified outside the lock**
`calibration_tracker.py:104` — Cache eviction runs after `with self._lock:` exits. Concurrent `get_weight_multiplier` calls can cause `RuntimeError: dictionary changed size during iteration`.
Fix: move eviction inside the lock block.

#### Security (14)

**M-33 — load_dotenv(~/.claude/.env) injects agent credentials into the trading process**
`config.py:9` — Trading bot unconditionally reads the Claude Code agent's personal credential store. `ANTHROPIC_API_KEY` and other secrets appear in any crash dump or env-logging code path.
Fix: remove the `~/.claude/.env` call; load credentials only from project-scoped sources.

**M-34 — Telegram Bot Token leaked to application log via exception message**
`notifier.py:204` — Token is embedded in the URL path. When `requests.post()` raises, `str(e)` from urllib3 includes the full URL. `logger.warning(..., e)` writes it verbatim.
Fix: `safe_msg = str(e).replace(self._telegram_token, "[REDACTED]")`

**M-35 — Unvalidated SENTRY_DSN — exception data sent to attacker-controlled host**
`sentry_init.py:8` — All captured exceptions (including executor stack frames with trade details) are sent to whichever host the DSN points to, without host validation.
Fix: validate the DSN host against the sentry.io domain before calling `sentry_sdk.init`.

**M-36 — SSRF via unvalidated SXBET_API_BASE_URL**
`sxbet_api.py:16` — Module-load-time assignment with no `assert_public_url` call, unlike the already-protected `gemini_api.py`, `gas_monitor.py`, `notifier.py`.

**M-37 — URL path injection via unsanitized subreddit and sort parameters**
`reddit_api.py:225` — `subreddit="poly/../user/me"` traverses to a different Reddit endpoint with the OAuth token attached.
Fix: allowlist `sort` to `{"hot","new","top","rising"}`; validate `subreddit` against `^[A-Za-z0-9_-]+$`.

**M-38 — Silent fallback to "YourApiKeyToken" in Polygonscan API**
`polygonscan_api.py:86` — When `POLYGONSCAN_API_KEY` is unset, the client uses the free-tier placeholder with no warning. Misconfiguration is invisible until a rate-limit spike.

**M-39 — Unguarded ["price"] subscript in SX Bet scan raises unhandled KeyError**
`scans/sxbet.py:55` — Every other scan module uses `.get()` with a numeric default. SX Bet uses bare subscript.
Fix: `float(bids[0].get("price") or 0)`

**M-40 — float() cast on Matchbook odds without try/except**
`scans/matchbook.py:56` — Non-numeric strings like `"suspended"` pass the `odds and` guard but crash `float()`.
Fix: `try: odds_f = float(odds); except (TypeError, ValueError): continue`

**M-41 — Index access on outcomePrices without length check (resolution.py security)**
`scans/resolution.py:173` — Same root cause as M-22; confirmed independently from a security-input-validation angle.

**M-42 — Unvalidated int() on Polygonscan fields can freeze block-cursor permanently**

**M-43 — Dashboard password exposed as plain CLI argument**
`scripts/go_live_check.py:88` — `--password` is visible in `ps aux`, shell history, and CI logs.
Fix: read from `DASHBOARD_PASS` env var; document `--password` as unsafe for production.

**M-44 — Dockerfile runs as root (no USER instruction)**
`Dockerfile:43` — Any RCE via a dependency runs as UID 0, enabling container-escape techniques.

**M-45 — Prompt injection via externally-sourced market question titles in LLM judge**
`market_discovery.py:275` — Market titles from Polymarket/Kalshi APIs are embedded verbatim in the Claude judge prompt. A crafted title can steer the judge to mark non-equivalent pairs as equivalent, producing guaranteed-loss arb trades.

**M-46 — No upper bound on POST Content-Length — post-auth DoS**
`dashboard.py:402` — An authenticated client can send `Content-Length: 1073741824` and stream it slowly, blocking the single-threaded HTTP server and halting all trading-state queries.
