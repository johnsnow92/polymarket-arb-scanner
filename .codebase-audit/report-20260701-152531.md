#### H-01 — `_select_fees` inverts expected-value probability weights

**File:** `fees.py:34`  
**Evidence:**
```python
return prob_a * case_b_fees + (1.0 - prob_a) * case_a_fees
```
**Why it matters:** The EV fee formula is `P(A wins) * fees_if_A_wins + P(B wins) * fees_if_B_wins`. The code has the weights swapped: `prob_a` weights `case_b_fees` and `(1 - prob_a)` weights `case_a_fees`. Every cross-platform profit calculation using `FEE_MODEL='expected_value'` underestimates fees for the higher-probability outcome and overestimates them for the lower-probability one, producing incorrect `net_profit` and `net_roi` for all such opportunities.

**Fix:** `return prob_a * case_a_fees + (1.0 - prob_a) * case_b_fees`
