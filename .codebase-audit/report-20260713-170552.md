## Executive Summary

**Verdict:** The codebase is not safe for unattended production live-trading in its current state.

Two deadlocks and one critical alert-silencing bug can stop risk controls from firing at the worst moments. Eighteen high-severity issues span live-order fire on a `DRY_RUN=true` flag, SSRF in three separate modules, credential leakage into the trading process, and systematic wrong fee and price calculations that produce false-positive arbitrage signals. Forty-six low-severity issues round out the picture: dead code, log/terminal injection, minor resource leaks, and underspecified tests.

#### C-1 · Daily-loss 100% breach alert permanently silenced after 80% rate-limit window
**File:** `alerting.py:192`
**Dimension:** bugs

**Evidence:**
```python
if loss_ratio >= 1.0 and not self._loss_100_fired:
    self._loss_100_fired = True   # set BEFORE the alert call
    self._loss_80_fired = True
    return self.alert(AlertType.DAILY_LOSS_LIMIT, Severity.CRITICAL, ...)
```

**Why it matters:** Both the 80% and 100% alerts share the same `AlertType.DAILY_LOSS_LIMIT` key in the rate-limiter. If the 80% alert fired at T=0 (rate-limit window=300 s) and the 100% breach is detected at T=120 s, `self.alert()` is blocked and returns `False`. But `_loss_100_fired` was set to `True` before the call, so every subsequent check skips the branch permanently. The operator receives no critical alert for a full daily-loss breach — ever.

**Fix:** Gate the flag on the alert actually being sent:
```python
alerted = self.alert(AlertType.DAILY_LOSS_LIMIT, Severity.CRITICAL, ...)
if alerted:
    self._loss_100_fired = True
    self._loss_80_fired = True
return alerted
```

#### H-1 · `_select_fees` inverts probability weights in expected-value mode
**File:** `fees.py:34`
**Dimension:** bugs

**Evidence:**
```python
# Current (wrong): fees when B wins weighted by prob(A wins)
return prob_a * case_b_fees + (1.0 - prob_a) * case_a_fees
# Correct:
return prob_a * case_a_fees + (1.0 - prob_a) * case_b_fees
```

**Why it matters:** Every cross-platform, Betfair/Smarkets, and triangular strategy under `FEE_MODEL="expected_value"` computes systematically wrong expected fees. Some unprofitable trades will appear profitable and be executed; real arbs will be filtered out.

**Fix:** Swap the operands in the return statement.
