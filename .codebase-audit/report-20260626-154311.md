## HIGH Severity

### H-01 — Gross profit formula understates Layer 4 P&L by up to 100x

**Files:** `fees.py` lines 1319, 1382, 1441, 1462, 1525

```python
contracts = max(1, int(size / entry_price)) if entry_price > 0 else 1
gross_profit = size * (exit_price - entry_price)  # wrong: should use contracts
```

For a $10 position at entry_price=0.50, contracts=20. Correct gross profit: 20*0.10=$2.00. Code computes 10.0*0.10=$1.00. Error grows as entry_price falls. Affects `net_profit_imbalance`, `net_profit_news_snipe`, `net_profit_time_decay`, and both legs of `net_profit_correlated`. All Layer 4 strategies are systematically screened out as unprofitable.

**Fix:** Replace `gross_profit = size * (exit_price - entry_price)` with `gross_profit = contracts * (exit_price - entry_price)` in all four functions.

### H-02 — Deadlock in `RegionRouter.should_route_externally` (non-reentrant lock)

**File:** `latency_monitor.py` line 366

```python
def should_route_externally(self, platform):
    with self._lock:                          # acquires Lock
        best_region = self.get_best_region(platform)  # also tries to acquire Lock

def get_best_region(self, platform):
    with self._lock:                          # second acquisition -> deadlock
```

`self._lock` is a plain `threading.Lock()`. Any call to `should_route_externally` in a deployed context freezes the calling thread permanently.

**Fix:** Change `self._lock = threading.Lock()` to `threading.RLock()`, or extract shared read logic into a private helper.

### H-06 — Sharpe ratio SQL formula computes annualized volatility, not Sharpe

**File:** `scripts/analytics.py` line 106

**File:** `fees.py` line 34

### M-49 — `MarketMaker.refresh_quotes` double-places bid and ask without updating inventory
