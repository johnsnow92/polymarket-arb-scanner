## Executive Summary

**Verdict:** The codebase is not production-safe for live automated trading in its current state. The confirmed defect count is high and the severity distribution is alarming: two confirmed deadlocks (one in capital optimisation, one in latency routing) block threads permanently, fee calculations are systematically wrong due to inverted EV weights and a stale Smarkets unit, P&L accounting is broken for all back-all settlements, and the CLOB refinement stage that is supposed to remove false positives silently leaves them in place. Beyond correctness, the security posture has significant gaps: unvalidated proxy URLs enable SSRF across all seven platform clients, Gemini withdrawals lack address validation, the dashboard exposes trading intelligence without authentication when DASHBOARD_PASS is unset, and Telegram error handling leaks the bot token into logs.

### C-1 — Deadlock: TaxOptimizer.get_harvest_candidates acquires non-reentrant lock twice

**File:** `capital_optimizer.py:314`

```python
with self._lock:
    for position_id, current_value in positions.items():
        if pnl_pct < -threshold:
            rate = (
                TAX_LONG_TERM_RATE
                if self.is_long_term(position_id)  # also does `with self._lock:`
                else TAX_SHORT_TERM_RATE
            )
```

`TaxOptimizer.__init__` creates a `threading.Lock()` (non-reentrant). `get_harvest_candidates` holds that lock, then calls `is_long_term()` which tries to acquire the same lock. The second acquisition blocks forever. Any call to `get_harvest_candidates` permanently freezes the calling thread.

**Fix:** Change `self._lock = threading.Lock()` to `self._lock = threading.RLock()`, or extract a `_is_long_term_unsafe()` helper that operates without acquiring the lock.

### C-2 — Deadlock: RegionRouter.should_route_externally calls get_best_region while holding self._lock

**File:** `latency_monitor.py:372`

```python
with self._lock:                                   # line 366
    best_region = self.get_best_region(platform)   # line 372 — also does `with self._lock:`
```

`get_best_region` acquires `self._lock` at line 344. The outer lock is already held, so the inner acquisition blocks forever. Any thread calling `should_route_externally` when `GEOGRAPHIC_LATENCY_ENABLED=True` is permanently deadlocked.

**Fix:** Extract inner logic of `get_best_region` into `_get_best_region_locked()` that expects the lock to be held; or convert to `threading.RLock`.

### H-1 — EV fee weights reversed in _select_fees — all cross-platform profit calculations wrong when FEE_MODEL='expected_value'

**File:** `fees.py:34`

```python
prob_a = max(0.0, min(1.0, price_a))
return prob_a * case_b_fees + (1.0 - prob_a) * case_a_fees  # WRONG — weights swapped
```

When side A wins (probability `prob_a`), `case_a_fees` apply. The weights are reversed. Affects all pairs using `net_profit_cross_betfair`, `net_profit_cross_smarkets`, `net_profit_cross_generic`, `net_profit_triangular`.

**Fix:** `return prob_a * case_a_fees + (1.0 - prob_a) * case_b_fees`

### H-2 — _calc_realized_pnl hardcodes $1 payout — all back-all settlement P&L is wrong

**File:** `continuous.py:262`


For a $10 arb costing $9.60 total, returns `1.0 - 9.60 = -$8.60` instead of `+$0.40`. All Betfair, Smarkets, SX Bet, and Matchbook back-all settlements call this path. The daily-loss limit and P&L database are corrupted.
