# Arbgrid Codebase Audit Report

**Date:** 2026-07-09
**Scope:** Full Python codebase, 239 files
**Audit method:** 3-lens adversarial verification; all findings below survived all three votes
**Total confirmed findings:** 130 (including 5 duplicate reports of 4 unique bugs noted inline)

---

## Executive Summary

**Verdict: High risk — do not enable live execution until critical and high findings are resolved.**

The codebase is architecturally sound and feature-complete for a 29-strategy trading system. However the audit uncovered two production-halting deadlocks, a fee-calculation formula whose operands are transposed (corrupting profit figures across all EV-mode scans), and two scan modules where CLOB refinement — the gate between mid-price false positives and real tradeable opportunities — is silently inoperative. The CI pipeline has a bash pipe that masks pytest exit codes, meaning test failures can silently be reported as passing.

## Critical Findings

### C-1. Deadlock: TaxOptimizer.get_harvest_candidates re-acquires non-reentrant lock via is_long_term

**File:** capital_optimizer.py lines 311-312
**Dimension:** bugs

self._lock is threading.Lock() (non-reentrant). get_harvest_candidates holds self._lock and calls self.is_long_term(position_id), which also acquires self._lock. The same thread blocks forever on the second acquisition. Any call to get_harvest_candidates() hangs the scanner permanently.

**Fix:** Change self._lock = threading.Lock() to self._lock = threading.RLock() in TaxOptimizer.__init__. Alternatively, inline the date comparison inside get_harvest_candidates without delegating to is_long_term.

(Reported twice at lines 311 and 312 — same code site.)

---

### C-2. Deadlock: RegionRouter.should_route_externally re-acquires non-reentrant lock via get_best_region

**File:** latency_monitor.py line 372
**Dimension:** bugs

Identical pattern to C-1. should_route_externally acquires self._lock and then calls self.get_best_region(platform), which tries to acquire it again. Any call to should_route_externally() deadlocks permanently.

**Fix:** Change self._lock = threading.Lock() to self._lock = threading.RLock() in RegionRouter.__init__.

## High Findings

### H-1. _select_fees expected-value formula has operands transposed

**File:** fees.py line 34
**Dimension:** bugs

Current: return prob_a * case_b_fees + (1.0 - prob_a) * case_a_fees
Should be: return prob_a * case_a_fees + (1.0 - prob_a) * case_b_fees

Every cross-platform, triangular, and multi-leg scan using FEE_MODEL='expected_value' gets a systematically wrong expected fee. This is one of the most widespread correctness bugs in the codebase.

**Fix:** Swap the multipliers.

---

### H-2. matchbook_api.py: _RateLimitError from circuit-open check propagates uncaught

**File:** matchbook_api.py line 112
**Dimension:** bugs

_request() has no @retry decorator. When the circuit breaker opens, _RateLimitError is raised before the try/except that catches requests.RequestException. It propagates uncaught through fetch_all_events, fetch_event_markets, list_runners, etc., crashing the scan thread.

**Fix:** Add @retry decorator to _request() with retry_if_exception_type((_RateLimitError, requests.ConnectionError, requests.Timeout)).

---

### H-3. sxbet_api.py: same uncaught _RateLimitError pattern

**File:** sxbet_api.py line 121
**Dimension:** bugs

Identical class of bug to H-2. All six public methods propagate the exception uncaught.

**Fix:** Same as H-2.

### H-4. Per-strategy loss-streak alert fires only on exactly 3 consecutive losses, silent for 4+

**File:** alerting.py line 324
**Dimension:** bugs

if losses == 3 should be if losses >= 3. After the 3rd loss triggers the alert, the 4th through 100th consecutive losses all go undetected. The global check_loss_streak function correctly uses >=.

**Fix:** Change if losses == 3: to if losses >= 3:.

### H-5. _refine_cross_all_with_clob: refined_out declared but never used — unprofitable Stage-1 candidates never filtered

**File:** scans/cross.py line 621
**Dimension:** bugs

refined_out = [] is declared, nothing appended, never returned, caller does not capture a return value. Stage-1 mid-price candidates that fail the CLOB ask-price check are never removed from the opportunities list. They reach filter_dust and the executor as real opportunities, producing false positives that can trigger unprofitable trades.

**Fix:** Build and return refined_out, update scan_cross_all to replace its list with the filtered result.

### H-6. triangular.py CLOB refinement always uses other_price=0

**File:** scans/triangular.py line 415
**Dimension:** bugs

scan_triangular never writes _price_a or _price_b to the opportunity dict. _refine_triangular_with_clob reads other_price = o.get("_price_b", 0) — always 0. With total_cost = pm_ask + 0 < 1.0, the >= 1.0 guard never fires. Every PM-involved triangular candidate passes refinement regardless of actual profitability.

**Fix:** Store _price_a and _price_b in the opportunity dict when building it in scan_triangular.

### H-7. triangular.py CLOB refinement reads yes_ask when Polymarket is on the NO side

**File:** scans/triangular.py line 421
**Dimension:** bugs

scan_triangular never writes _side_a or _side_b. The else-branch defaults pm_side="yes" and reads clob["yes_ask"] even when Polymarket holds the NO position. Combined with H-6, both leg prices and fee attribution are wrong.

**Fix:** Add _side_a and _side_b to the opportunity dict in scan_triangular.

### H-8. correlated.py: CLOB fetch maps wrong market dict to key when market_b is the long leg

**File:** scans/correlated.py line 367
**Dimension:** bugs

When price_b < price_a, _long_market = market_b but the fetch key is _market_key_a. After CLOB resolution the long and short CLOB data are fully swapped, giving wrong ask prices and depth for roughly half of all correlated pairs.

**Fix:** Key on _long_leg/_short_leg: (opp.get("_long_leg"), opp.get("_long_market")).

### H-9. imbalance.py: Stage-2 collapse check ignores direction flips

**File:** scans/imbalance.py line 82
**Dimension:** bugs

The collapse check compares abs(current_ratio) < collapse_threshold but never verifies sign consistency. A ratio that flips from +0.6 (buy YES) to -0.45 (buy NO) passes the magnitude check and a trade is placed in the wrong direction.

**Fix:** if abs(current_ratio) < collapse_threshold or (original_ratio > 0) != (current_ratio > 0): continue

_token_ids only captures the BUY leg (then_market). There are no _if_token_ids for the SELL leg. The executor would lack the token identifier to place the sell order, either failing or silently reducing the trade to a naked directional bet.

### H-11. CI: pytest exit code masked by pipe — test failures can silently pass

**File:** .github/workflows/test.yml line 32
**Dimension:** bugs

In bash without set -o pipefail, the pipeline exit code is tee's exit code (always 0). If pytest crashes internally before printing its summary line, the grep-based check misses it and the workflow exits 0, falsely reporting "All tests passed."

**Fix:** Capture PIPESTATUS[0] after the pipeline, before re-enabling set -e.

### H-15. SSRF via unvalidated SXBET_API_BASE_URL environment variable

**File:** sxbet_api.py line 16
**Dimension:** security

SXBET_API_URL = os.getenv("SXBET_API_BASE_URL", "https://api.sx.bet") is used without validation. gemini_api.py guards its equivalent with assert_public_url(). An attacker who can write a Railway env var can redirect all SX Bet traffic to http://169.254.169.254/.

**File:** capital_optimizer.py line 71
**Dimension:** bugs

**Fix:** return float("inf") if roi > 0 else roi when days <= 0.

## Medium Findings

### Execution and Runtime Bugs

M-1. Kalshi WebSocket cache stores yes_ask/no_ask but revalidation reads yes_price/no_price
ws_feeds.py:435 / executor.py:1016 / continuous.py:488 — Keys mismatch; cached_k.get("yes_price") always returns None. Every WS-triggered revalidation falls back to a synchronous REST call, adding latency and risking rate limits.
Fix: Standardize on one key convention across all three files.

M-2. _fill_price never stored in legs during concurrent execution — partial-fill hedger always sizes from limit price
executor.py:2800 — _execute_legs (sequential) correctly stores leg["_fill_price"] = fill_price. _execute_legs_concurrent omits this. Hedge is sized from the original limit price regardless of actual fill.
Fix: Add leg["_fill_price"] = fill_price after results[idx] = True in _execute_legs_concurrent.

M-3. sxbet_api.py and matchbook_api.py: HTTP 429 increments the circuit-breaker failure counter
sxbet_api.py:133 / matchbook_api.py:119 — Three consecutive rate-limit responses trip the circuit breaker and block all requests for 30 seconds, compounding the rate-limit problem.
Fix: Skip _circuit.record_failure() for 429; add brief sleep and return None.

M-4. betfair_api.py: six methods bypass self._request() and its retry/circuit-breaker
betfair_api.py:206 — list_markets, list_runners, get_current_orders, get_balance, get_order_status, cancel_orders use self.session.post() directly. Circuit breaker state is incoherent.
Fix: Route all six methods through self._request() as place_orders was fixed to do.

M-5. kalshi_api.py: _circuit.record_success() called for all non-429 responses including 4xx/5xx
kalshi_api.py:166 — HTTP 500 responses reset the failure counter; circuit breaker never opens on server-level failures.
Fix: if resp.status_code < 400: _circuit.record_success() elif resp.status_code != 429: _circuit.record_failure()

M-6. gemini_api.py: event_type determined from contracts key but raw_contracts falls back to markets key
gemini_api.py:314 — When contracts appear under markets, event_type sees empty list and assigns categorical. Binary events with a single contract generate no recognized arb opportunity.
Fix: Compute raw_contracts first with the fallback, then derive event_type from raw_contracts.

M-7. Platform price included in signal-aggregator consensus used to measure divergence from that price
event_monitor.py:271 — Line 271 adds platform_price to the aggregator under the platform's own name. Consensus is pulled toward platform_price, systematically underestimating divergence.
Fix: Remove line 271. Only external signals should form the consensus.

M-8. Prometheus histogram double-counts bucket values
metrics.py:78 — observe() stores cumulative counts; get_prometheus_text() additionally sums them into a running total, double-counting. /metrics endpoint produces invalid histograms.
Fix: Add break after entry["buckets"][b] += 1 in observe() to store only the lowest matching bucket.

M-9. Backtest total_cost wrongly includes resolution/convergence target price
backtest.py:197 — For ResolutionSnipeOpp, snapshot.py stores price_b=1.0 (payout target, not a cost). Backtest computes total_cost = trade_price + 1.0, halving computed units and simulated P&L.
Fix: Add per-strategy early exits: for directional strategies, total_cost = price_a only.

M-10. credential_health.py: check_all_platforms awaits coroutines sequentially instead of concurrently
credential_health.py:78 — Awaiting in a plain for-loop runs them one at a time. With 8 platforms x 20s timeout each, one check round can take 160 seconds.
Fix: await asyncio.gather(*[self._check_platform_health(p) for p in self.clients], return_exceptions=True)

M-11. _read_cached_price in cross_pair_index.py accepts entries when _ts is 0 or missing
cross_pair_index.py:71 — if ts short-circuits when ts=0, skipping the age check. An entry with _ts=0 is returned as fresh regardless of actual age.
Fix: Remove the if ts guard: if time.time() - ts > max_age: return None

M-12. calibration_tracker.py: cache invalidation runs outside the lock
calibration_tracker.py:103 — Cache keys are deleted after self._lock is released. Concurrent reader can see stale entry; concurrent dict mutation can raise RuntimeError.
Fix: Move cache invalidation inside the with self._lock: block.

M-13. _refine_cross_all_with_clob: r2 fee uses single Stage-1 other_price for alternative direction
scans/cross.py:663 — The alternative direction (r2) reuses e.g. K_NO price where K_YES is needed. If best_r resolves to r2, opportunity dict has fabricated profit figure.
Fix: Only evaluate r1 (the already-chosen direction) during CLOB refinement.

M-14/15. scan_kalshi_binary and scan_kalshi_multi: future.result() not wrapped in try/except
scans/kalshi.py:173 and :300 — Any network error during depth fetch aborts the entire scan, returning zero results despite valid Stage-1 candidates.

M-16. supabase_sync.py: int(None) crash when timestamp key present with NULL value
supabase_sync.py:49 — int(row.get('timestamp', 0)) returns int(None) when the key exists with NULL value, crashing sync_reward_metrics.

M-17. scans/sxbet.py: bare dict key access on bid/ask entries raises unhandled KeyError
scans/sxbet.py:55 — All four bids[0]["price"] / asks[0]["price"] accesses use direct indexing. A missing key crashes both scan functions.

M-18. time_decay.py: guaranteed_gain computed with operands reversed — always negative
scans/time_decay.py:81 — guaranteed_gain = buy_below_price - target_price is negative whenever consensus_prob > buy_below_price, the normal case.

M-19. correlated.py: Layer-4 spread-floor gate body is pass — never drops any opportunity
